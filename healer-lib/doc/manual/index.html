 (require healer-lib/start) package: healer

Provides a generic start function for other healer games to build upon.

syntax

(start avatar (food ...) (friends ...) (enemies ...))

The basic starter for a healing game. It is a macro because we want people to type:

(start cat (dog apple))

... not:

(start cat (list dog apple))

So we capture the user’s code, package the parameters into lists and call whatever function has been "bound" with bind-start-to. This allows different variations on the healing game to be congifured by defining a function (e.g. my-start) and binding it:

(bind-start-to my-function)

The function should be one that takes between 0 and 4 parameters. In most cases, it will directly call generic-start, which does the heavy lifting for the game.

procedure

(generic-start [#:bg bg    
  #:avatar-sprite avatar    
  #:starvation-rate starvation-rate    
  #:food-sprites fs    
  #:npc-sprites ns    
  #:enemy-sprites es    
  #:score-prefix score    
  #:instructions instructions]) → void?
  bg : bg? = (custom-bg #:rows 2 #:columns 2)
  avatar : described-sprite? = (list question-mark-icon)
  starvation-rate : number? = 25
  fs : (list-of described-sprite?) = '()
  ns : (list-of described-sprite?) = '()
  es : (list-of described-sprite?) = '()
  score : string? = "Friends Healed"
  instructions : entity? = (make-instructions default-text)

value

described-sprite? : 
(or/c sprite?
      (list sprite? color-string?)
      (list sprite? number?)
      (list sprite? color-string? number?))
Contract for recognizing things that are valid avatar, foods, friends, or enemies in the healer language.

 (require healer-lib/examples-lib) package: healer

Convenience functions for auto-generating katas from a healing game language.

syntax

(define-food-examples
             #:lang lang
             #:start START
             #:avatars (AVATAR-A AVATAR-B AVATAR-C AVATAR-D)
             #:foods   (FOOD-A FOOD-B FOOD-C FOOD-D)
             #:rand  RAND)
Produces various food katas from pre-written templates, using the provided identifiers. The kata stimuli is also generated automatically.
